Controle de Fluxo de Execução

Introdução

  • Usados na execução de ações diferentes em conformidade com os dados manipulados pelos algoritmos.
  • São estruturas básicas de controle:
    • Seqüenciais;
    • De Decisão;
    • De Repetição.

Estrutura Seqüencial

  • Cada comando é executado na seqüência exata em que compõem os algoritmos.
  • Exemplo:
algoritmo CalculaMedia;
variáveis
   N1,
   N2,
   Media : real;
início
   leia (N1, N2);
   Media <- (N1 + N2) / 2;
   escreva (Media);
fim.

Estrutura de Decisão (SE)

  • O fluxo de execução dos comandos de um algoritmo é determinado pelos dados, conforme o atendimento de condições.
  • Estrutura SE:
se <condição> então
   <comando1>
senão
   <comando2>
  • Onde o comando identificado por <comando1> será executado se <condição> for V e <comando2>, se falsa.
  • O senão é opcional, ou seja, pode ser omitida, isso quando somente o <comando1> interessa para a resolução implementada pelo algoritmo.

Estrutura de Decisão (CASO)

  • Estrutura CASO:
caso <condição> seja
  <resultado1> : <comando1>;
  <resultado2> : <comando2>;
  ...
  <resultadoN> : <comandoN>;
senão
  <comandos>
fim;
  • Na execução deste, o comando, ou bloco de comandos, <comandoN> será efetuado se <resultadoN> atender à <condição> dada:
  • Trata-se de um seletor de opções.
  • O(s) comando(s) indicados por <comandos> somente será executado se nenhum dos resultados elencados atender à condição.
  • Exemplo: Algoritmo que efetua a leitura de um número correspondente a um mês do ano e exibe o nome deste por extenso.
algoritmo Identificadores;
variáveis
   Mes : inteiro;
início
   leia (Mes);
   caso Mes seja
      1: escreva ('Janeiro');
      2: escreva ('Fevereiro');
      3: escreva ('Março');
      4: escreva ('Abril');
      5: escreva ('Maio');
      6: escreva ('Junho');
      7: escreva ('Julho');
      8: escreva ('Agosto');
      9: escreva ('Setembro');
     10: escreva ('Outubro');
     11: escreva ('Novembro');
     12: escreva ('Dezembro');
   senão
      escreva ('Mês inexistente');
   fim;
fim.

Estruturas de Repetição

  • Como sugere a denominação, são úteis à repetição de comando, ou bloco de comandos. Exemplo: calculadora, útil à realização de inúmeros cálculos.
  • São denominados laços ou loops
  • São estruturas de repetição:
    • Para
    • Enquanto
    • Repita

Estrutura de Repetição (PARA)

  • Efetua as repetições com o auxílio de um contador.
  • É usada quando se sabe previamente o número de repetições a serem efetuadas.
  • Estrutura PARA
para <Contador> <- <Início> até <Fim> faça
   <comando>;
  • Onde:
    • Contador - variável tipo inteiro de controle do número de repetições.
    • Início - valor de inicialização do contador.
    • Fim - valor final do contador.
    • Comando - comando ou bloco de comandos a serem executados repetidas vezes.
  • Exemplo:
algoritmo MaiorDeTrês;
variáveis
   I, N, Maior : inteiro;
início
   Maior <- 0;
   para I <- 1 até 3 faça
      início
         leia (N);
         se N > Maior então
            Maior <- N
      fim
   escreva (Maior)
fim.
  • Neste serão lidos 3 números (o bloco de comandos em negrito serão executados 3 vezes) e será exibido o maior deles.
  • Importante destacar que o incremento do contador é de uma unidade, a cada passo, e automático.
  • As repetições ocorrem enquanto o contador não atinge o valor estipulado como Fim.
  • Exemplo: algoritmo que efetua a soma dos 20 primeiros números pares maiores que zero.
algoritmo SomaPares;
variáveis
   N, Soma : inteiro;
início
   Soma <- 0;
   para N <- 1 até 20 faça
      Soma <- Soma + 2 * N;
   escreva (Soma);
fim.

Estrutura de Repetição Condicional (ENQUANTO)

  • Os comandos subordinados à estrutura em estudo são executados enquanto uma determinada condição seja satisfeita.
  • Estrutura ENQUANTO
enquanto <Condição> faça
   <comando>;
  • Quando a execução do algoritmo alcança a estrutura ENQUANTO a condição é testada. Se o resultado for falso os comandos subordinados à estrutura são ignorados, caso contrário, são efetuados e (re)efetuados enquanto a condição seja verdadeira.
  • Quando os comandos subordinados ao ENQUANTO estiverem em execução, somente a mudança de estado da condição provoca o término do processo.
  • Caso a lógica dos comandos da estrutura não altere o estado da condição, a execução será infinita. É o que chamamos de loop (ou laço) infinito.
  • Exemplo: algoritmo que calcula o fatorial de um dado número:
algoritmo Fatorial;
variáveis
   Resultado, Numero : inteiro;
início
   Resultado <- 1;
   leia (Numero);
   enquanto (Numero > 1) faça
      início
         Resultado <- Resultado * Numero;
         Numero <- Numero - 1;
      fim;
   escreva (Resultado);
fim.
  • Exemplo: algoritmo que efetua a soma dos 20 primeiros números pares maiores que zero.
algoritmo SomaPares;
variáveis
   N, Soma : inteiro;
início
   Soma <- 0;
   N <- 1;
   enquanto N < 21 faça
      início
         Soma <- Soma + 2 * N;
         N <- N + 1;
      fim;
   escreva (Soma);
fim.

Estrutura de Repetição Condicional (REPITA)

  • Os comandos subordinados à estrutura em estudo são executados até que uma determinada condição seja satisfeita.
  • Estrutura REPITA
repita
  <comandos>;
até <condição>
  • Os comandos subordinados ao REPITA são executados pelo menos uma vez, pois a condição é testada somente no final da estrutura.
  • O funcionamento desta é similar ao da estrutura ENQUANTO, exceto quanto à posição em que a condição é testada.
  • Exemplo: algoritmo para exibir os 25 termos e o somatório da PG: 3, 9, 27, ...
algoritmo ProgressaoGeometrica;
variáveis
   Contador,
   Termo,
   Soma : integer;
início
   Contador <- 1;
   Termo <- 3;
   Soma <- 0;
   repita
      escreva (Termo);
      Soma <- Soma + Termo;
      Termo <- Termo * 3;
      Contador <- Contador + 1;
   até Contador > 25;
   escreva (Soma);
fim.
  • Exemplo: algoritmo que efetua a soma dos 20 primeiros números pares maiores que zero.
algoritmo SomaPares;
variáveis
   N, Soma : inteiro;
início
   Soma <- 0;
   N <- 1;
   repita
      Soma <- Soma + 2 * N;
      N <- N + 1;
   até N > 20;
   escreva (Soma);
fim.
