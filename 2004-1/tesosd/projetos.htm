Entrega

  • O projeto deve ser entregue em disquete ou CD no momento da apresentação.

Apresentação

  • Cada grupo terá no máximo 15 minutos para apresentar o projeto.
  • A apresentação consiste em demonstrar todas as funcionalidades do sistema através de exemplos práticos.
  • Não é necessário apresentar o código-fonte.

Ambiente de Testes/Implantação

  • Foram instalados no servidor Machadinho os seguintes softwares
    • JBoss 3.2.3
    • SQL Server 2000 (Enterprise Edition)
  • As aplicações devem ser instaladas e configuradas nesse servidor.

Base de dados

  • Como alguns projetos têm bases de dados em comum, foram criados 3 bancos de dados (DCCE1, DCCE2 e DCCE3) com o mesmo conteúdo.
  • A base de dados já tem as informações básicas preenchidas.
  • O esquema de banco de dados foi documentado usando o DB Designer 4, estando disponível também o dicionário de dados.
  • Para acessar o servidor, utilize o usuário "usr_dcce" e a senha "usr_dcce", o qual tem permissão de acesso aos 3 bancos de dados.
  • Utilize o Enterprise Manager ou SQL Query Analizer para acessar o BD. Caso o cliente não esteja instalado na máquina, o servidor de terminais Tucurui dispõe dos mesmos instalados.

JBoss

  • O JBoss foi instalado como serviço e estará sempre rodando.
  • O servidor foi configurado para receber as requisições HTTP (Web) na porta 8081.
  • Para fazer o deployment das aplicações, utilize o compartilhamento jboss_deploy de Machadinho, o qual dá acesso ao diretório da configuração default do JBoss. Não foi possível dar permissão de acesso a alguns alunos porque não tinham conta cadastrada no domínio discentes.
  • Foram criados 3 Data Sources (DCCE1, DCCE2 e DCCE3) no JBoss, os quais dão acesso aos 3 Bancos de Dados (DCCE1, DCCE2 e DCCE3).

Entity Beans

  • Foram criados e implantados em Machadinho 3 módulos EJB com Entity Beans iguais, um para cada banco de dados (DCCE1, DCCE2 e DCCE3). Há diferenças no Deployment Descriptor de cada um para que tenham nomes JNDI diferentes.
  • Para ter acesso aos detalhes dos componentes, baixe o projeto EntityBeansDCCE.zip . Esses componentes só podem ser modificados pelo professor.
  • Para o desenvolvimento das aplicações, baixe o projeto EntityBeansDCCETeste.zip, o qual contém uma classe de teste para acessar o Entity Bean Aluno. Esse projeto serve de base para as aplicações. Existe um arquivo chamado BeansDCCE.jar, o qual contém as interfaces Home, Remota, Home Local e Local, que deve ser adicionado ao projeto caso não utilize o baixado como base.
  • Para acessar um componente utilize o seguinte padrão: <nome-do-banco>.<nome-do-componente>. Para o componente Aluno do Banco DCCE2, o nome JNDI é DCCE2.Aluno. Esse prefixo é importante identificar a base de dados na qual o componente está persistindo os dados.
