Projetos

Visão Geral

  • O objetivo do projeto é desenvolver uma aplicação completa, inclusive com armazenamento de arquivo e interface gráfica com o usuário (GUI).
  • Dentre as sugestões de projeto, mas não limitando-se a elas, temos:
    • Cadastro de Professores
    • Cadastro de Disciplinas
    • Cadastro de Grade Curricular de Cursos
    • Cadastro de Oferta Semestral de Disciplinas
    • Jogo de Dominó
    • Jogos de Baralho
    • Editor de Texto
    • Planilha Eletrônica
    • Chat/Mensagens
    • ...

Requisitos não funcionais

  • A linguagem de programação a ser usada é Java.
  • O ambiente de desenvolvimento usado fica a critério do aluno.
  • A interface deve ser Swing.
  • A persistência deve ser em arquivo.

Grupos

  1. Os grupos devem ser formados por até 4 (quatro) membros.
  2. Todos os membros do grupo devem conhecer profundamente o projeto

Critérios de Avaliação

Critério Peso Observação

Interface e Usabilidade

3
  • Gráfica baseada em componentes Swing
  • O funcionamento do programa deve ser de fácil entendimento
  • A utilização do programa deve ser fácil
Eficácia e Eficiência

3
  • Aplicação cumpre os requisitos funcionais e não funcionais
Legibilidade e qualidade do código

3
  • Legibilidade e qualidade do código escrito em Java
Documentação/Instalação/Configuração

1
  • O código do programa deve ser bem documentado
  • A instalação e configuração também deve ser documentada

Avaliação

  1. Será feita a partir dos critérios acima.
  2. Como todos os membros do grupo devem conhecer profundamente o projeto, durante a apresentação do projeto, o professor irá indicar quais componentes devem explicar quais partes do projeto.
  3. A nota final dos alunos de um grupo será igual, sendo assim, um aluno que não souber explicar como uma determinada funcionalidade está implementada irá prejudicar os demais.
